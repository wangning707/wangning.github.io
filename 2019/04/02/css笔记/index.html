点透事件(移动端)

事件的触发时间按由早到晚排列为：touchstart 早于 touchend 早于 click。
亦即click的触发是有延迟的，这个时间大概在300ms左右。
由于我们在touchstart阶段就已经隐藏了罩层A，当click被触发时候，能够被点击的元
素则是其下的B元素，由于B绑定了click事件（或者B本身默认存在click事件），所以
B的click事件被触发，产生了点透的情况。

A为遮罩层，设置了点击消失之类的事件，B在遮罩层下面且B本身有点击事件，就会发生点透事件。
阻止点透事件：
1、github上有一个fastclick可以完美解决
引入fastclick.js，因为fastclick源码不依赖其他库所以你可以在原生的js前直接加上
window.addEventListener( “load”, function() {
FastClick.attach( document.body );
}, false );
当然require的话就这样：
var FastClick = require(‘fastclick‘);
FastClick.attach(document.body, options);
2、preventDefault()，阻止事件默认触发
$(“#aa”).on(“touchend”, function (event) {
//很多处理比如隐藏什么的
event.preventDefault();
});
3、延迟处理事件
$(“#aa”).on(“tap”, function (event) {
setTimeout(function(){
//很多处理比如隐藏什么的
},320);
});

canvas与svg的区别

canvas绘制的是位图，canvas可以看作是一个画布，是标量图，因此可以在canvas中引入jpg或者png这类格式的图片，许多大型网络游戏都是用canvas画布做出来的。
canvas的图片不可以被引擎抓取。
canvas通常是通过js来实现，

svg最初是使用xml技术的描述二维图形的语言，它只能绘制矢量图，不能引入普通的图片，因为矢量图不会失真的效果，一般svg用来做一些小图标，而且它也非常适合来做地图，百度地图就是使用svg绘制出来的。
svg可以被引擎抓取，支持绑定事件。
svg通过标签来实现。

CSS性能优化

现在的浏览器实现动画的方式一般有4种，改变位置，改变大小，旋转，改变透明度。
当下做动画最好的几个属性

  • opacity
  • translate
  • rotate
  • scale

    层，重绘，回流，重布局，图层重组的概念

    浏览器渲染页面时，会将页面分为许多个图层，每一个图层上有一个或者多个节点
    渲染流程:
    1、获取dom后分割图层
    2、对每个图层的节点计算样式结果(recalculate style-样式重计算)
    3、为每个节点生成图形和位置(layout-回流和重布局)
    4、将每一个节点绘制填充到图层位图中(重绘)
    5、图层作为纹理上传到GPU
    6、叠加多个图层到页面上生成屏幕图像(composite layers-图层重组)

chrome中满足以下条件即可创建图层:

  • 3D或透视变换（perspective transform）CSS属性
  • 使用加速视频解码的\
  • 拥有3D（WebGL）上下文或加速的2D上下文的\节点
  • 混合插件（如Flash）
  • 对自己的opacity做CSS动画或使用一个动画webkit变换的元素
  • 拥有加速CSS过滤器的元素
  • 元素有一个包含复合层的后代节点（一个元素拥有一个子元素，该子元素在自己的层里）
  • 元素有一个z-index较低且包含一个复合层的兄弟元素（换句话说就是该元素在复合层上面渲染）

需要注意的是，如果图层中某个元素需要重绘，那么整个图层都需要重绘。比如一个图层包含很多节点，其中有个gif图，gif图的每一帧，都会重回整个图层的其他节点，然后生成最终的图层位图。所以这需要通过特殊的方式来强制gif图属于自己一个图层（translateZ(0)或者translate3d(0,0,0)），CSS3的动画也是一样（好在绝大部分情况浏览器自己会为CSS3动画的节点创建图层）

层和CSS动画

简化一下上述过程，每一帧动画浏览器可能需要做如下工作：

  1. 计算需要被加载到节点上的样式结果（Recalculate style–样式重计算）
  2. 为每个节点生成图形和位置（Layout–回流和重布局）
  3. 将每个节点填充到图层中（Paint Setup和Paint–重绘）
  4. 组合图层到页面上（Composite Layers–图层重组）

如果我们需要使得动画的性能提高，需要做的就是减少浏览器在动画运行时所需要做的工作。最好的情况是，改变的属性仅仅印象图层的组合，变换（transform）和透明度（opacity）就属于这种情况

现代浏览器如Chrome，Firefox，Safari和Opera都对变换和透明度采用硬件加速

修改透明度不会触发重绘

GPU在绘图时修改透明度属性只需要改变已经画好的图层纹理的alpha值，并不需要重绘。

transform

当我们通过transform修改节点的位置、大小、旋转等，不会触发重布局，而 left&top属性会触发重排;

JS动画与CSS动画

JS动画

  • 缺点：JavaScript在浏览器的主线程中运行，而其中还有很多其他需要运行的JavaScript、样式计算、布局、绘制等对其干扰。这也就导致了线程可能出现阻塞，从而造成丢帧的情况。

  • 优点：JavaScript的动画与CSS预先定义好的动画不同，可以在其动画过程中对其进行控制：开始、暂停、回放、中止、取消都是可以做到的。而且一些动画效果，比如视差滚动效果，只有JavaScript能够完成

CSS动画

  • 缺点：缺乏强大的控制能力。而且很难以有意义的方式结合到一起，使得动画变得复杂且易于出问题。
  • 优点：浏览器可以对动画进行优化。它必要时可以创建图层，然后在主线程之外运行。
